# 보안 (SECURITY)

## 설계상 보안 특성

- **로컬 전용**: 수집 데이터는 당신의 기기를 떠나지 않습니다(예외: PRIVACY.md의 "외부 통신").
- **웹 화면 바인딩 제한**: `kit.py web`은 인증이 없으므로 `127.0.0.1`(루프백)에만 바인딩합니다.
  `HOST`를 루프백 밖 주소로 지정하면 서버가 **기동을 거부**합니다. 원격 접근이 필요하면
  SSH 터널이나 인증을 갖춘 리버스 프록시를 앞단에 두세요.
- **원문 폴더 차단**: `source/`·`quarantine/`·`accounts/` 이하 전체는 검색과 직접 조회
  (`vault.get_note`) 양쪽에서 차단됩니다(`kitlib/security.py`의 `BLOCKED_FOLDER_PREFIXES`).
- **시크릿 마스킹**: 반환 텍스트의 비밀번호·API 키·OTP·인증 링크·사설망 주소는 마스킹됩니다.
- **API 키 취급**: LLM enrich용 키는 **환경변수로만**(`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`)
  읽습니다. 웹 폼이나 `config.json`에 저장하지 않습니다. `config.json`은 `.gitignore` 대상입니다.

## 사용자 책임

- 볼트에는 암호화되지 않은 개인정보가 저장됩니다 — 디스크 암호화(FileVault)를 켜고,
  공유/동기화 경로에 두지 마세요.
- `katok`·`imsg` 등 서드파티 CLI는 각 프로젝트의 신뢰성을 직접 확인하고 설치하세요.

## 취약점 제보

보안 이슈를 발견하면 공개 이슈로 올리기 전에 저장소 소유자에게 비공개로 알려 주세요.
재현 절차와 영향 범위를 포함해 주시면 감사하겠습니다.
